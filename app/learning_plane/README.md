# Learning Plane / Institutional Adaptation Governance

## Overview
The Learning Plane provides the canonical governance layer for converting signals, incidents, and near-misses into validated institutional learning. It enforces a strict discipline: merely declaring "we learned a lesson" or archiving a postmortem is insufficient. The plane requires a verifiable chain from signal detection through root cause validation, lesson extraction, target updates, and anti-recurrence validation.

## Core Concepts

* **Signals -> Findings -> Lessons -> Updates -> Validation -> Recurrence Control -> Trust Flow**: This is the mandatory lifecycle for learning.
* **Why Postmortem != Institutional Learning**: A postmortem is a document. True learning requires actionable updates to the system's constitution, policy, state, or contracts, backed by anti-recurrence validation.
* **No Buried Lessons**: Lessons extracted must map to explicit update targets. Orphaned lessons are tracked as Learning Debt.
* **No Action Graveyard**: Actions without adoption and anti-recurrence validation are treated as false confidence and lower the learning trust verdict.
* **Precedent and Recurrence**: Stale precedents, normalization of workarounds, and same-class recurrences actively degrade the learning posture and can block system progression.

## Integration
This plane integrates deeply with the rest of the system (Constitution, Policy, Assurance, Release, Change, State) to ensure that learning is not an isolated documentation exercise but a driver of institutional adaptation.
