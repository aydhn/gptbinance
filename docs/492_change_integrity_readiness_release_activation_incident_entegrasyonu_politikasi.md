# Change Integrity Readiness; Release, Activation, Incident Entegrasyonu Politikası

- **Change_Integrity Domain:**
  Sistemin change truth omurgasıdır; request clarity, approval legitimacy, rollback readiness, verification discipline sağlar.

- **Release / Activation / Incident / Migration Integrations:**
  - *Release Plane:* Change evidence taşımayan rollouts güvenli sayılmaz.
  - *Activation:* Governed change ve independent verification olmadan bir sonraki aşamaya geçirilemez.
  - *Incident Plane:* Emergency değişiklikler kalıcılaştırmak üzere takip edilmelidir.

- **Policy Obligations:**
  Eksik onaylı, rollback desteksiz değişiklikler policy evalutation aşamasında "Review" veya "Deny" ile engellenir.

- **Evidence Graph / Review Packs:**
  Süreçler arası kanıtlar, artefact ve review packs olarak kaydedilir.
