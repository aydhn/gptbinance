# Phase 117 - Liability Plane Architecture

## Harms/Actions -> Causation/Fault -> Allocation/Exposure -> Indemnity/Trust Flow
The liability plane ensures that any action resulting in harm is explicitly evaluated. The flow requires identifying causes, mapping them to fault structures, allocating the resulting exposure, checking for indemnities, and producing a trust verdict.

## Why Cause != Liability
Causality represents physical or logical sequence (who triggered what). Liability represents the allocated legal, operational, or financial responsibility for the consequences. An actor can cause an event without being liable (e.g., executing a properly authorized, but flawed, procedure).

## Why No Blame Shifting
The liability plane enforces that liability cannot be silently reassigned. Any change in liability requires an explicit causation reassessment, an indemnity execution, or a validated exoneration.

## Liability Governance Logic
The core principle is that every liability must be accounted for. Shared liability must be explicitly distributed, residual exposure must be tracked even when caps apply, and no settlement is considered final if unaddressed residual liability remains.
