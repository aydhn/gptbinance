# Phase 127 Definition of Done

## Completion Criteria
1. The Insolvency Plane framework is fully instantiated with typed models.
2. The canonical insolvency registry separates estates, claims, priorities, stays, plans, liquidations, and deficits.
3. Debt, equivalence, and class impairment visibility are operational.
4. CLI commands correctly query all insolvency layers.
5. No silent "estate leakage", "false secured comfort", or "hidden residual deficit" can exist without a warning being generated.

## Intentionally Deferred
- Real-time ingestion of real-world bankruptcy dockets.
- Fully automated algorithmic distribution engine (human trustees remain responsible for execution).

## Transition to Phase 128
The system is ready for Phase 128 (Recovery-Driven Allocation Governance) once the fundamental insolvency truth substrate is validated and its test suite passes.
