# Phase 135: Accountability Plane & Consequence Governance Architecture

## Overview
Accountability is not merely assigning blame. The Accountability Plane enforces a typed, canonical registry where:
`Subject -> Duty -> Breach -> Sanction -> Restitution -> Finality`
are strictly linked.

## Why no scapegoating?
System explicitly rejects "symbolic sanctions" (e.g., assigning a minor warning for a material breach to artificially close an accountability loop). The `AccountabilityTrustEngine` guarantees that consequences are proportional and mapped to actual duty holders.
